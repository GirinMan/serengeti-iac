# GIS Facility Data Recovery: 좌표 틀어짐 현상

현재 포천시 하수관로 시설 데이터는 좌표계 데이터 변환이 되어 있는데,
실제 지도에 렌더링된 모습을 보면 일반적인 도로의 형태 등과 약간씩 틀어져 있는 것 같다.
좌표 변환 기록에 문제가 없는지 스크린샷을 찍어 LLM의 이미지 분석 기능으로 직접 확인하자.
필요하면 좌표 변환 및 마이그레이션 프로세스를 개선하고, GIS에 사용되는 전체 데이터베이스를 초기화하고 다시 마이그레이션하자.

오케이, 이건 그냥 “살짝 어긋남” 수준이 아니라
👉 **거의 100% 좌표 변환 과정에서 축/부호/원점 문제**입니다.

특히 네가 표시한 것처럼

> “상하가 뒤집힌 느낌 (선대칭처럼)”
> 이건 아주 전형적인 패턴이에요.

---

## 🔥 가장 유력한 원인 3개 (우선순위 순)

### 1️⃣ Y축 반전 (좌표계 방향 문제)

GIS 좌표계는 보통:

* Y ↑ = 북쪽

근데 어떤 데이터/엔진은:

* Y ↓ = 아래쪽 (이미지 좌표계 방식)

👉 이 상태에서 그대로 쓰면
→ **위아래가 완전히 뒤집힘 (거울 반사 느낌)**

✔ 지금 이미지랑 거의 동일한 증상

---

### 2️⃣ 위도/경도 순서 + 변환 혼합 오류

예를 들어:

```text
(위도, 경도) → (경도, 위도) 잘못 넣고
거기에 projection까지 하면
```

👉 단순 위치 이동이 아니라
→ **형태 자체가 뒤틀리거나 대칭처럼 보일 수 있음**

---

### 3️⃣ TM/UTM → WGS84 변환에서 축 처리 오류

한국에서 특히 흔한 케이스:

* EPSG:5179 (UTM-K / Korea 2000)
* EPSG:5181 (중부원점 TM)

👉 변환할 때:

* 축 순서 (x,y)
* false northing
* scale factor

이 중 하나 틀리면
→ **특정 축 기준으로 뒤집힌 형태 발생**

---

## 🧠 왜 “선대칭”처럼 보이냐?

지금 이미지 보면:

* 도로 패턴 자체는 유지됨
* 근데 **위/아래만 뒤집힘**

👉 이건 랜덤 오차가 아니라
👉 **수학적으로 y → -y 형태 변환이 들어간 것**

즉:

```text
원래: (x, y)
지금: (x, -y)
```

---

## 🔍 빠르게 검증하는 방법 (중요)

### 방법 1: 한 점만 찍어보기

예:

* 시청, 병원 같은 확실한 좌표 하나

👉 지도에 찍었을 때

* 위아래 반대로 나오면 = Y축 문제 확정

---

### 방법 2: 강제 뒤집기 테스트

코드에서:

```python
y = -y
```

또는

```python
lat = -lat
```

👉 이거 했을 때 정상으로 돌아오면
→ 100% 축 반전 문제

---

### 방법 3: QGIS에서 확인

* 레이어 추가
* “레이어 CRS 재정의 (Set Layer CRS)” vs “재투영 (Reproject)”

👉 여기서 헷갈리면 이런 문제 바로 발생함

---

## 💥 실전에서 가장 흔한 실수 패턴

너 상황 거의 이거일 가능성 큼:

1. 원본: EPSG:5179 (미터 좌표)
2. 근데 이걸:

   * 그냥 lat/lon처럼 처리했거나
   * 변환 라이브러리에서 축 순서 옵션 안 줌

특히 `pyproj`에서:

```python
Transformer.from_crs(..., always_xy=True)
```

이거 안 쓰면
👉 축 뒤집힘 발생 가능

---

## ✅ 해결 방법 (핵심만)

### ✔ pyproj 쓰면

```python
Transformer.from_crs("EPSG:5179", "EPSG:4326", always_xy=True)
```

---

### ✔ Leaflet / Mapbox면

* 반드시 lat, lon 순서 확인
* (lon, lat) 넣으면 뒤틀림 발생

---

### ✔ 근본 해결

👉 **데이터 CRS 정확히 확인 → 그에 맞게 변환**

---

## 🎯 결론

지금 증상은:

> ❌ GPS 오차 아님
> ❌ 단순 projection mismatch 아님
> ✅ **Y축 반전 or 축 순서 오류 (거의 확정)**
