# Harbor 2.11 LTS (Layer 1 Ops)

Self-hosted OCI registry. NPM terminates TLS and proxies
`https://${HARBOR_HOSTNAME}` → `harbor-nginx:8080` on the `proxy-tier` network.

## Files in this directory

- `harbor.yml` — configuration template with `${...}` env placeholders.
- `docker-compose.yml` — minimal viable Harbor 2.11 stack. Expects
  `./common/config/` to have been rendered by Harbor's `prepare` step.
- `README.md` — this file.

Not committed (generated by `install.sh` on the host):
- `./common/config/**` — rendered Harbor configs (nginx, core, db, etc.)
- `./data/**` — persistent state (registry blobs, postgres, redis, job logs)

## First-time bootstrap

1. **Set env vars in repo `.env`** (see `.env.example`):
   - `HARBOR_HOSTNAME=harbor.<CF_DOMAIN>`
   - `HARBOR_ADMIN_PASSWORD=<strong-secret>`
   - `HARBOR_DB_PASSWORD=<strong-secret>`
   - `CF_HARBOR_HOST=harbor.<CF_DOMAIN>`

2. **Render `harbor.yml` with env values** (strips `${...}` placeholders):

   ```bash
   cd docker/layer1-ops/harbor
   set -a; . ../../../.env; set +a
   envsubst < harbor.yml > harbor.yml.rendered
   mv harbor.yml.rendered harbor.yml.local
   ```

3. **Download Harbor 2.11 offline installer** once into a scratch dir:

   ```bash
   cd /tmp
   HARBOR_VER=v2.11.1
   curl -LO "https://github.com/goharbor/harbor/releases/download/${HARBOR_VER}/harbor-offline-installer-${HARBOR_VER}.tgz"
   tar xf "harbor-offline-installer-${HARBOR_VER}.tgz"
   ```

4. **Run `prepare` only** (we use our own compose file, not the installer's):

   ```bash
   cp docker/layer1-ops/harbor/harbor.yml.local /tmp/harbor/harbor.yml
   (cd /tmp/harbor && sudo ./prepare --with-trivy)
   cp -r /tmp/harbor/common docker/layer1-ops/harbor/
   # Enable relative URLs so the registry generates scheme-less Location
   # headers (proxy supplies HTTPS). Without this, docker push follows
   # http:// URLs and fails with 401 / "authorize header needed".
   sudo sed -i '/^http:/a\  relativeurls: true' docker/layer1-ops/harbor/common/config/registry/config.yml
   ```

   This renders `./common/config/**` next to our `docker-compose.yml`.

5. **Start via Makefile**:

   ```bash
   make harbor
   ```

6. **Add NPM proxy host** for `${HARBOR_HOSTNAME}` → `harbor-nginx:8080`
   (HTTP forward, enable Websockets, disable "Force SSL" — Cloudflare / NPM
   above handle TLS). Advanced config must include
   `proxy_set_header X-Forwarded-Proto https;` so Harbor generates https
   Location headers, and `client_max_body_size 0;` for large image pushes.
   See `scripts/setup_npm_hosts.py` (Harbor entry).

7. **Add Cloudflare Tunnel ingress** for `${HARBOR_HOSTNAME}` → NPM.

## Post-bootstrap admin steps

1. **Login** to `https://${HARBOR_HOSTNAME}` as `admin` /
   `${HARBOR_ADMIN_PASSWORD}`.

2. **Create project `girinman`** (public: false). This matches the image
   naming convention `harbor.<CF_DOMAIN>/girinman/<app>:<tag>`.

3. **Create a robot account** under the `girinman` project for CI pushes:
   - Project → Robot Accounts → `+ New Robot Account`
   - Name: `ci-push` (Harbor will prefix with `robot$girinman+`)
   - Permissions: `push` and `pull` on `repository`; `push` on `artifact`
   - Expiration: 365 days (rotate annually)
   - Copy the generated secret immediately — Harbor will not show it again.
   - Store as GitHub Actions secrets `HARBOR_ROBOT_USER` and
     `HARBOR_ROBOT_TOKEN` in the app repos.

4. **Verify**:

   ```bash
   curl -s https://${HARBOR_HOSTNAME}/api/v2.0/health | jq .
   # expect {"status":"healthy",...}
   ```

## Upgrading

Bump image tags in `docker-compose.yml` (all `goharbor/*:vX.Y.Z` together),
re-run `prepare` with the matching installer version to regenerate
`./common/config/`, then `make harbor`.

## Notes

- Registry blobs live under `./data/registry` (bind-mount). For large usage,
  move this subdir to `${ARCHIVE_STORAGE_ROOT}` and replace with a symlink
  — same pattern Minio uses.
- Harbor's `harbor-log` container collects syslog from siblings on
  `tcp://127.0.0.1:1514`; nothing is published beyond loopback.
- Trivy adapter pulls vuln DB on first start (~30s). Disable by removing the
  `trivy-adapter` service and `--with-trivy` flag above if unused.
