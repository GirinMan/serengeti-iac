# Perplexity Search Log — 2026-03-23

## Query 1: Cloudflare Tunnel public hostnames via CLI/API

- **Background**: NPM proxy hosts were created via API script, but Cloudflare Tunnel public hostnames still need to be added manually in the Zero Trust dashboard. Looking for programmatic alternatives.
- **Query**: "How to programmatically add Cloudflare Tunnel public hostnames via CLI or API instead of the dashboard? Can cloudflared CLI or Cloudflare API create ingress rules and DNS records for a tunnel automatically?"
- **Time**: 2026-03-23 ~02:00 KST
- **Result Summary**:
  - `cloudflared tunnel route dns <TUNNEL> <HOSTNAME>` auto-creates DNS CNAME records
  - Ingress rules are defined in a local YAML config file, not via the `route dns` command
  - Cloudflare API endpoints: `/accounts/:account_id/tunnels` for tunnel management, `/zones/:zone_id/dns_records` for DNS
  - Locally-managed tunnels use config file + CLI; remotely-managed tunnels (token-based) use dashboard or API
- **Sources**:
  - https://developers.cloudflare.com/cloudflare-one/networks/connectors/cloudflare-tunnel/do-more-with-tunnels/local-management/tunnel-useful-commands/
  - https://developers.cloudflare.com/cloudflare-one/networks/connectors/cloudflare-tunnel/routing-to-tunnel/
  - https://developers.cloudflare.com/cloudflare-one/networks/connectors/cloudflare-tunnel/do-more-with-tunnels/local-management/create-local-tunnel/
