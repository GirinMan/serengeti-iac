## Search 1

- Time: 2026-03-10 21:20 KST
- Background:
  - Ghost가 현재 설계의 PostgreSQL 조합에서 안정적으로 올라오지 않아 대체 가능한 콘텐츠 플랫폼 후보가 필요했다.
  - 조건은 self-hosted, 유지보수 용이성, 확장성, 콘텐츠 관리 API 제공, PostgreSQL 친화성이다.
- Query:
  - `Compare self-hosted headless CMS options that officially support PostgreSQL and provide content management APIs for a maintainable, extensible blog/content platform. Prioritize maintenance, extensibility, self-hosting, and official Postgres support. Candidates: Directus, Strapi, Payload CMS, KeystoneJS, ApostropheCMS. Return concise ranking with official source URLs.`
- Result summary:
  - 후보군은 `Directus`, `Strapi`, `Payload CMS` 중심으로 압축됐다.
  - `Strapi`는 범용성과 플러그인 생태계가 강점으로 반복 언급됐다.
  - `Directus`는 SQL DB 위에 바로 관리 UI와 REST/GraphQL API를 얹는 방식이 강점으로 보였다.
  - `Payload`는 성능과 코드 기반 확장성에서 강하다는 비교 결과가 다수 노출됐다.
- URLs surfaced:
  - `https://dev.to/chrdek/the-top-8-headless-cms-with-self-hosting-options-13jn`
  - `https://prismic.io/blog/best-headless-cms-for-developers`
  - `https://hygraph.com/blog/directus-alternative`

## Search 2

- Time: 2026-03-10 21:21 KST
- Background:
  - 1차 후보 중에서 기존 인프라 계층의 PostgreSQL을 재사용하고 운영 복잡도를 줄일 대상을 좁히기 위해 공식 문서 도메인 위주로 비교가 필요했다.
- Query:
  - `Using official documentation only, compare Directus, Strapi, and Payload CMS for a self-hosted homelab content platform that must use PostgreSQL, expose content APIs, remain maintainable, and be extensible for future apps. Evaluate PostgreSQL support, API model, admin/editor UX, extensibility, and operational complexity. Recommend the best fit and runner-up.`
- Result summary:
  - 응답은 `Payload` 쪽 공식 비교/문서 비중이 높게 나왔지만, 그 안에서도 세 가지 차이가 드러났다.
  - `Payload`는 코드 기반 확장성과 PostgreSQL adapter, migration/Drizzle 흐름이 강점이다.
  - `Directus`는 기존 SQL DB 위에 관리 UI와 API를 즉시 얹는 운영 모델이 장점으로 비교됐다.
  - `Strapi`는 범용적이지만 운영과 초기 셋업 복잡도가 상대적으로 높게 언급됐다.
  - 현재 이 리포의 목적은 “서비스 전용 신규 DB 추가 없이 기존 인프라 PostgreSQL 위에 콘텐츠 관리 API를 얹는 것”이므로 최종 선택은 `Directus`, 차선은 `Payload CMS`로 정리했다.
- URLs surfaced:
  - `https://payloadcms.com/posts/blog/performance-benchmarks`
  - `https://payloadcms.com/compare/directus`
  - `https://payloadcms.com/compare/strapi`
  - `https://payloadcms.com/docs/database/postgres`
  - `https://payloadcms.com/posts/blog/payload-2-0`

## Decision

- Selected: `Directus`
- Why:
  - 기존 `PostgreSQL` 인프라 계층을 그대로 사용 가능
  - 콘텐츠 관리 UI와 REST/GraphQL API를 즉시 제공
  - 별도 서비스 전용 DB 엔진을 추가하지 않아도 됨
  - 향후 `Custom API`, 자동화, 외부 앱이 같은 콘텐츠 백엔드를 재사용하기 쉬움
- Runner-up:
  - `Payload CMS`
  - 이유: PostgreSQL adapter와 코드 기반 확장성은 우수하지만, 현 리포 기준으로는 Directus보다 운영자 즉시 사용성과 UI 중심 유지보수성이 떨어진다고 판단

## Search 3

- Time: 2026-03-10 21:31 KST
- Background:
  - 사용자가 `Directus`는 엄밀히 말해 블로그가 아니라고 지적했다.
  - 따라서 “개발/취미 블로그 자체”에 적합한 self-hosted 플랫폼을 다시 골라야 했다.
- Query:
  - `For a self-hosted developer/hobby blog on a homelab, compare alternatives to Ghost that are actually suitable as a blog platform, not just a generic data CMS. Prioritize maintainability, extensibility, self-hosting, content editing UX, and availability of APIs for future integrations. Consider WordPress, WriteFreely, Payload CMS, Strapi, ApostropheCMS, and Directus. Recommend the best fit and runner-up, and explain why some options are better for CMS/data-platform use than for a real blog.`
- Result summary:
  - `WordPress`가 일반적인 self-hosted 블로그 플랫폼으로 가장 적합한 방향으로 반복 노출됐다.
  - `WriteFreely`는 미니멀한 글쓰기 경험과 self-hosting 관점에서 차선책으로 적합해 보였다.
  - `Payload`, `Strapi`, `ApostropheCMS`, `Directus`는 블로그 플랫폼보다는 CMS/데이터 플랫폼 성격이 강하다고 정리됐다.
  - 메인 작업 판단으로는 “실제 블로그”는 `WordPress`, “범용 콘텐츠 API”는 필요 시 `Directus` 재검토가 맞다.
- URLs surfaced:
  - `https://inblog.ai/blog/ghost-alternative-blogs`
  - `https://www.blogbowl.io/blog/posts/top-10-ghost-alternatives-for-bloggers-newsletters-in-2025`
  - `https://hyvor.com/blog/ghost-alternatives`
  - `https://alternativeto.net/software/ghost/?license=opensource`

## Updated Decision

- Blog platform:
  - Selected: `WordPress`
  - Why:
    - 일반적인 개발/취미 블로그에 가장 자연스럽다.
    - 성숙한 편집 UX, 테마/플러그인 생태계, 내장 REST API가 있다.
    - 장기적으로 문서형 사이트, 포트폴리오, 간단한 서비스 페이지까지 함께 확장하기 쉽다.
- Minimal alternative:
  - `WriteFreely`
  - 이유: 글쓰기 집중형 미니멀 블로그에는 좋지만, 확장성과 범용 운영 생태계는 WordPress보다 약하다.
- Content/data CMS:
  - `Directus`는 “블로그 자체”보다는 범용 콘텐츠 관리 플랫폼 용도로만 보류
